    <h3 class="crt-green mt-2">Hard Skills: The Metal and the Wires</h3>
    <p>Because apparently knowing how to do the job is still part of the job.</p>

    <h4 class="crt-green mt-2">Programming Languages</h4>
    <ul class="list--custom">
        <li><strong>Level 1:</strong> Can read, write, and debug in one or two approved dialects of human despair (JavaScript, Python, etc.) with supervision.</li>
        <li><strong>Level 2:</strong> Proficient across multiple syntaxes. Writes maintainable code instead of artisanal chaos.</li>
        <li><strong>Level 3:</strong> Speaks fluent machine. Enforces standards, establishes conventions, and judges others silently in code reviews.</li>
    </ul>

    <h4 class="crt-green mt-2">SQL / Database Development</h4>
    <ul class="list--custom">
        <li><strong>Level 1:</strong> Writes basic queries and occasionally remembers WHERE.</li>
        <li><strong>Level 2:</strong> Optimizes queries, uses indexes without breaking production, and knows GROUP BY isn’t a social club.</li>
        <li><strong>Level 3:</strong> Designs query systems so efficient they almost justify the cost of the license. Teaches others the ancient ways of the WITH clause.</li>
    </ul>

    <h4 class="crt-green mt-2">Transaction Safety / Rollback</h4>
    <ul class="list--custom">
        <li><strong>Level 1:</strong> Uses transactions (usually).</li>
        <li><strong>Level 2:</strong> Writes rollback logic before running updates (a true rebel).</li>
        <li><strong>Level 3:</strong> Defines standards for safety, preaches data sanctity like a priest of atomic consistency.</li>
    </ul>

    <h4 class="crt-green mt-2">Data Modeling / Relational Design</h4>
    <ul class="list--custom">
        <li><strong>Level 1:</strong> Understands relationships...at least between tables.</li>
        <li><strong>Level 2:</strong> Builds normalized models that don’t implode under moderate use.</li>
        <li><strong>Level 3:</strong> Architects the ERP data cathedral and ensures referential integrity like it’s a religion.</li>
    </ul>

    <h4 class="crt-green mt-2">Reporting / BI Tools</h4>
    <ul class="list--custom">
        <li><strong>Level 1:</strong> Can open SSRS without crying.</li>
        <li><strong>Level 2:</strong> Creates and maintains reports that don’t explode on parameter change.</li>
        <li><strong>Level 3:</strong> Defines enterprise BI standards and dashboards that make executives feel “data-driven.”</li>
    </ul>

    <h4 class="crt-green mt-2">ETL / Data Integration</h4>
    <ul class="list--custom">
        <li><strong>Level 1:</strong> Moves CSVs around like a courier.</li>
        <li><strong>Level 2:</strong> Builds reliable pipelines and handles exceptions with grace (and sometimes try/catch).</li>
        <li><strong>Level 3:</strong> Automates integrations so seamless you forget they exist, until they fail at 3 a.m.</li>
    </ul>

    <h4 class="crt-green mt-2">Source Control / Deployment</h4>
    <ul class="list--custom">
        <li><strong>Level 1:</strong> Uses Git like a glorified Dropbox.</li>
        <li><strong>Level 2:</strong> Manages branches and pull requests responsibly.</li>
        <li><strong>Level 3:</strong> Masters the pipeline. Deploys with confidence and a healthy amount of paranoia.</li>
    </ul>
